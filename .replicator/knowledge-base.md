# Knowledge Base

## Backend
- Use CommonJS modules in JavaScript.
- Prefer async/await over promise chains in controllers and service layers.
- Keep validation close to the request entry point.

## Database
- Use snake_case for SQL tables and columns.
- Add schema changes through standalone migration files.
- Prefer soft-delete markers over physical deletes.

## Frontend
- Keep HTML semantic and readable.
- Use BEM-style class names when adding CSS.
- Avoid inline styles in generated HTML unless the framework requires them.
